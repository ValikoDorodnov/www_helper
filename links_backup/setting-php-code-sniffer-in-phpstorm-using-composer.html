Настройка PHP Code Sniffer в PhpStorm с помощью Composer

Для начала нужно что бы путь к директории с интерпретатором PHP был прописан в переменной окружения path и установлен Composer.

Примечание: для того что бы PhpStorm подтянул изменения path может потребоваться завершение сеанса пользователя.

Далее с помощью Composer установим squizlabs/php_codesniffer глобально. 

composer global require squizlabs/php_codesniffer

В настройках PhpStorm установим путь к phpcs, для этого перейдем в Settings > Languages & Frameworks > PHP > Code Sniffer.

Узнать расположение исполняемого файла phpcs можно с помощью команд 

composer config --global home
composer config --global bin-dir

Для Windows 7 это обычно 

C:\Users\%userName%\AppData\Roaming\Composer\vendor\bin\phpcs.bat

В настройках PhpStorm включим проверку с помощью phpcs, для этого перейдем в Settings > Editor > Inspections > PHP > PHP Code Sniffer validation

Ссылки:

  • https://getcomposer.org/doc/00-intro.md#installation-linux-unix-osx
  • https://github.com/squizlabs/PHP_CodeSniffer
